# タスク計画

## 元の要求
`/kiro:spec-impl texas-holdem-review-fixes-1 3` — タスク3「ショーダウン時のポット配分の正当性確保」の実装

## 分析結果

### 目的
単一ポットの前提で、ショーダウン時の配分後にいかなるプレイヤーのチップも負にならないようにする。オールインが複数いる場合の配分ロジックを確認し、必要なら修正する。仕様として単一ポット・簡略化配分であることを明記する。（要件4）

### スコープ
対象ファイル: `src/domain/showdown.ts`（57行）、`src/domain/showdown.test.ts`（492行）

### 要件ごとの変更要否判定

| 要件 | 判定 | 根拠 |
|------|------|------|
| 単一ポット・簡略化配分の仕様明記 | **変更不要** | `showdown.ts:28-30` に「本バージョンではサイドポットを実装せず、単一ポットで均等配分する」「端数は最初の勝者に加算」「不変条件: chips >= 0 かつ配分合計 = pot」のコメントが既に存在。`resolveUncontestedPot` にも同様のコメントあり（46-47行） |
| 配分後の chips >= 0 保証 | **変更不要** | `showdown.ts:39` で `p.chips + share + extra` と加算のみ実行。`share = Math.floor(pot / winners.length)` は非負、`extra = remainder = pot - share * winners.length` も非負。chips に対する減算操作が存在しないため、構造的に chips が負になる経路はない |
| 複数オールイン時の配分ロジック | **変更不要** | `showdown.ts:36-41` の配分はプレイヤーの chips 残高に依存せず、ポット全額を勝者に均等分配する。テスト `showdown.test.ts:307-335` で「chips=0 のオールイン複数プレイヤー」、`showdown.test.ts:338-374` で「chips=0 の単独勝者」のケースが検証済み |
| チップ保存則の維持 | **変更不要** | テスト内で `calcTotalChips(result)` による保存則チェックが4箇所で実施済み（`showdown.test.ts:240-241`, `301-302`, `334-335`, `466-467`） |
| `resolveUncontestedPot` の正当性 | **変更不要** | `showdown.ts:48-56` で勝者に `p.chips + state.pot` と加算のみ。テスト `showdown.test.ts:447-468` で chips=0 の勝者ケース、`showdown.test.ts:470-490` で pot=0 のケースを検証済み |

### 実装アプローチ

**このタスクでは実装変更は不要です。**

現在の `showdown.ts` は以下をすべて満たしています：

1. **仕様の明記**: コメントで単一ポット・サイドポット非対応・端数処理・不変条件を明記済み
2. **chips >= 0 の保証**: 配分ロジックは加算のみで、減算操作がなく構造的に安全
3. **チップ保存則**: `share * winners.length + remainder = pot` が数学的に保証
4. **エッジケースのテストカバレッジ**: オールイン複数、chips=0 勝者、pot=0、奇数ポット、3-way/4-way タイがすべてテスト済み

テストも追加不要です。既存の `showdown.test.ts` に17テストケースがあり、要件4の全受け入れ基準をカバーしています。
