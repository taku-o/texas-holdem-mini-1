# 決定ログ

## 1. 統合テストのイテレーション回数を5回に制限
- **背景**: 当初「人間が全CPUを倒すシナリオ」で200回のハンドイテレーションを設定したが、テスト実行がタイムアウト（OOM/exit code 137）で失敗した
- **検討した選択肢**: 200回、50回、10回、5回
- **理由**: 5回でもゲームフローの正常動作（複数ハンド連続・チップ増減・プレイヤー除外）を十分検証でき、テスト実行時間を2〜3秒以内に抑えられるため

## 2. テストファイルを3ファイルに分割
- **背景**: 統合テスト・ショーダウンエッジケース・除外エッジケースを1ファイルにまとめると責務が混在する
- **検討した選択肢**: 1ファイルに全テスト集約、2ファイル（統合+単体）、3ファイル（統合・ショーダウン・除外）
- **理由**: 1ファイル1責務の原則に従い、テスト対象の関心事ごとに分割。既存テストも `handEvaluator.test.ts`、`cpuStrategy.test.ts` 等で責務分離されている慣習に合わせた

## 3. ゲーム再開テストは統合テストファイルに含める
- **背景**: ゲーム再開フロー（ゲーム終了後の `startGame()` 再呼び出し）を独立ファイルにするか統合テストに含めるか
- **検討した選択肢**: 独立ファイル `gameRestart.test.ts`、`gameIntegration.test.ts` 内に含める
- **理由**: ゲーム再開はゲーム開始〜終了の統合フローの延長であり、同じセットアップ（gameSetup → gameEngine）を共有するため、統合テストファイルに含めるのが自然
