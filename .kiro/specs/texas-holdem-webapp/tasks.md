# Implementation Plan

## Implementation Tasks

- [x] 1. プロジェクト・フロントエンド基盤の構築
- [x] 1.1 (P) React と TypeScript を用いたブラウザ向けプロジェクトを初期化し、ビルド・開発サーバーを動作させる
  - データベースやサーバー永続化は使わず、クライアントのみで動作する構成とする
  - _Requirements: 3.1, 3.2, 3.3_

- [x] 1.2 (P) Tailwind CSS を導入し、Apple 風の余白・タイポグラフィの土台を用意する
  - _Requirements: 4.1_

- [ ] 2. ゲーム用データ型と定数の定義
- [ ] 2.1 カード・プレイヤー・ゲーム状態・役ランク・プレイヤーアクションを表す型と、初期チップ数などの定数を定義する
  - 参加者6（ディーラー1・プレイヤー5）、プレイヤーは人間1・CPU4の構成を型で表現できるようにする
  - ゲーム状態はブラウザメモリのみで保持する想定で設計する
  - _Requirements: 1.1, 1.2, 2.1, 3.3_

- [ ] 3. 役判定の実装
- [ ] 3.1 (P) @pokertools/evaluator を用いて、7 枚のカードから最良の 5 枚役のランクを返すアダプタを実装する（役はハイカード〜ロイヤルストレートフラッシュ）
  - 同一役の場合はキッカー等で比較可能なランクを返す
  - _Requirements: 5.4, 5.5_

- [ ] 4. ゲームエンジンの中核ロジック
- [ ] 4.1 ゲーム開始時に人間プレイヤーの席をランダムに 1 席決め、全プレイヤーに初期チップを付与した状態を生成する処理を実装する
  - _Requirements: 1.3, 2.1, 8.1_

- [ ] 4.2 各ハンドでディーラー左隣を SB・その左を BB としてブラインドをポストし、各プレイヤーにホールカード 2 枚を配る処理を実装する
  - _Requirements: 5.1, 5.2_

- [ ] 4.3 ベッティングラウンドでフォールド・チェック・ベット・コール・レイズを受け付け、ルールに沿って検証し状態を更新する処理を実装する（オールインは許容するがポットは単一とする）
  - _Requirements: 5.3_

- [ ] 4.4 ショーダウンで役判定を用いて勝者を決め、ポットを配分し、チップが 0 になったプレイヤーを除外または参加不可とする処理を実装する
  - _Requirements: 5.4, 2.2, 2.3_

- [ ] 4.5 1 ハンド終了後にディーラーボタンを次のプレイヤーに移し、必要に応じてブラインドを更新して次のハンドを開始する処理を実装する
  - _Requirements: 8.2_

- [ ] 4.6 ゲーム終了条件（人間のチップが 0、ユーザーが終了を選択、CPU が全員チップ 0 のいずれか）に達したら終了状態に遷移する処理を実装する
  - _Requirements: 8.3_

- [ ] 4.7 ディーラー業務（カード配布・ボタン移動・ポット集約と配分）が一連の流れで正しく行えるよう、上記処理を組み合わせる
  - _Requirements: 1.4_

- [ ] 5. CPU の行動決定ロジック
- [ ] 5.1 現在のゲーム状態とプレイヤー ID から、フォールド・チェック・コール・ベット・レイズのいずれかをルールに沿って自動で決定するロジックを実装する
  - ベット・レイズ時はチップ数も返す。初版はシンプルなルールベースでよい
  - _Requirements: 7.1_

- [ ] 6. ゲームコントローラーの実装
- [ ] 6.1 ゲーム状態を 1 つ保持し、ゲーム開始操作で参加者・席順・初期チップを設定して最初のハンドを開始する処理を実装する
  - _Requirements: 8.1_

- [ ] 6.2 人間プレイヤーが行ったアクション（チェック・ベット・コール・レイズ・フォールド）を受け取り、ゲームエンジンで検証・状態更新したうえで、必要なら CPU ターンを連続で進める処理を実装する
  - _Requirements: 6.1, 6.3, 1.4_

- [ ] 6.3 CPU のターンでは自動で行動を決定し、その結果を状態に反映してから次のプレイヤーまたはラウンドへ進める処理を実装する
  - _Requirements: 7.1, 7.3_

- [ ] 7. テーブル・カード・席の表示 UI
- [ ] 7.1 テーブル上にコミュニティカード（0〜5 枚）とポット額を表示し、カード・チップが視覚的に判別できるようにする
  - _Requirements: 4.2, 5.1, 5.2_

- [ ] 7.2 各席のプレイヤー（人間/CPU）・チップ数・手札（人間は自分の 2 枚を表示、CPU はショーダウン等のタイミングでのみ表示）・フォールド状態を表示し、人間の席を強調する
  - _Requirements: 1.1, 1.2, 2.2, 4.2, 7.2_

- [ ] 8. 人間用アクションバーの実装
- [ ] 8.1 人間のターン時のみ、チェック・ベット・コール・レイズ・フォールドを選択できる UI を表示する
  - _Requirements: 6.1_

- [ ] 8.2 ベット・レイズ選択時にチップ数を指定する手段（スライダー・入力欄・クイックベットのいずれか）を提供する
  - _Requirements: 6.2_

- [ ] 8.3 ルール上選択可能なアクションのみを有効にし、選択できないアクションは無効化する
  - _Requirements: 6.3_

- [ ] 9. ゲーム画面ルートと開始・終了・再開の UI
- [ ] 9.1 ゲーム未開始時は「ゲーム開始」などの入口を表示し、ゲーム中はテーブル・席・アクションバーを配置して状態を子に渡す
  - _Requirements: 4.1, 4.3, 8.1_

- [ ] 9.2 ゲーム終了時（人間チップ 0、ユーザーが終了選択、CPU 全員チップ 0 のいずれか）は「新しいゲームを始める」などの再開手段を表示する
  - _Requirements: 8.3_

- [ ] 10. 結合・動作確認とテスト
- [ ] 10.1 ゲーム開始から 1 ハンドの進行（プリフロップ〜ショーダウン）、人間のアクション・CPU の自動行動・ポット配分・次ハンド・ゲーム終了までが一通り動作することを確認する
  - _Requirements: 1.1, 1.2, 1.3, 1.4, 2.1, 2.2, 2.3, 5.1, 5.2, 5.3, 5.4, 5.5, 6.1, 6.2, 6.3, 7.1, 7.2, 7.3, 8.1, 8.2, 8.3_
